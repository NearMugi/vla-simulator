# AntiGravity 開発指示書: vla-simulator

`README.md` に記載されたシステム構成に基づき、本プロジェクトの開発を開始してください。 
開発にあたっては、以下のガイドラインとステップに従って、**動作確認が取れた状態を積み上げながら**進めてください。

## 0. 開発方針

* **インクリメンタル開発:** 各ステップで「何が動くべきか」を確認し、成功してから次へ進むこと。
* **疎結合の維持:** Frontend (React), Robot Control (ROS 2), Backend (FastAPI) のインターフェースを意識し、互いに依存しすぎないコードを書くこと。
* **シミュレーション優先:** まずはブラウザ上のシミュレータと ROS 2 の同期を完璧にすること。

---

## 1. 開発ロードマップ（推奨ステップ）

開発の優先順位は、システム全体のボトルネックとなる「通信」と「ロボット制御」から開始することを推奨します。

### Step 1: 制御基盤の構築 (Frontend ⇔ ROS 2 疎通) [DONE]

* **目標:** ブラウザからボタンを押し、WSL2上の ROS 2 Node へ信号が届くことを確認する。
* **タスク:**
    * Vite + React の初期化。
    * WSL2 での `rosbridge_suite` 起動。
    * `roslibjs` を使用し、ブラウザからダミーの座標を ROS 2 トピックとしてパブリッシュする。

### Step 2: 物理エンジンとロボットモデルの同期 [DONE]

* **目標:** 指定された URDF を読み込み、Rapier3D（ブラウザ）と MoveIt2（ROS 2）で同じアームを表示・同期させる。
* **タスク:**
    * URDF の配置と読み込みロジックの実装。
    * MoveIt2 による逆運動学（IK）計算の有効化。
    * ROS 2 から返ってきた関節角度を `KinematicBody` として Rapier3D のアームに反映する。

### Step 3: 推論サーバーの最小実装 (Backend ⇔ Frontend 疎通)

* **目標:** FastAPI サーバーを立て、ブラウザから送った画像に対してダミーの座標を返す。
* **タスク:**
    * FastAPI サーバーの Docker コンテナ化。
    * `/predict` エンドポイントの作成。
    * CORS 設定を適切に行い、ブラウザからの画像送信（HTTP POST）とレスポンス受信を確認する。

### Step 4: VLAモデルの統合とオンデマンド・ロード

* **目標:** 実際の OpenVLA モデルをロードし、推論結果でシミュレータのアームを動かす。
* **タスク:**
    * `bitsandbytes` による 4-bit 量子化ロードの実装。
    * `PEFT` を用いた LoRA 重みの動的ロード機能の実装。
    * シーケンス図に定義された「自律制御サイクル (Closed-loop)」のループ処理を実装。

### Step 5: 学習データ収集機能 (Training Mode)

* **目標:** 人間の操作ログを画像と共に保存し、データセット化する。
* **タスク:**
    * シミュレータ上の操作イベントのキャプチャ。
    * 画像とアクション（座標）をペアでバックエンドへ送信・保存するロジックの実装。

---

## 2. 実装にあたっての技術的指定

* **座標系:** すべて右手系、メートル(m)、ラジアン(rad)で統一すること。
* **エラー処理:** `rosbridge` の切断や推論 API のタイムアウト時に、アームの急激な挙動変化を防ぐセーフティを設けること。
* **環境変数:** `.env` を活用し、接続先 IP やモデルパスをハードコードしないこと。

## 3. 提出物

各ステップの完了ごとに、以下の出力を求む：
1.  動作を確認したコード一式。
2.  そのステップで作成したインターフェース（API 仕様やトピック名）のメモ。
3.  次のステップへ進むための前提条件が満たされているかの確認報告。
